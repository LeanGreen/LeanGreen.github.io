1. 搭建所需环境

1.1 Git

首先到这里下载git，然后逐步进行安装。

在开始菜单找到Git Bash右键属性，修改Git Bash的配置 : 将Git Bash设置为快速编辑模式, 可以更好的使用该命令行工具。

打开Git bash，执行：
ssh-keygen -t rsa -C "your_email@email.com"

该命令后面的邮箱就是GitHub的注册邮箱;

使用该命令之后, 会出现提示选择ssh-key生成路径, 这里直接点回车默认即可, 生成的ssh-key在默认路径中。

Overwrite (y/n)?则输入y再回车

Enter passphrase (empty for no passphrase)则输入密码，该密码以后每次提交博客更新时都会需要；然后在确认输入一次。

结果显示如下：

进入生成的ssh目录: C:\Users\用户.ssh中, 使用记事本打开id_rsa.pub文件, 将该文件中的内容复制;
例如：

ssh-rsa AAAAB3NzaC1yc2EAAAABIwAAAQEAtT1YCeaNulpfC+ARqAWrCdfpi6CpW3gkGT0hp6Q8by7NnEfy4dah9CwSrNbWJH5eS4tiqckE+bdbSVNvAboFD1MtGZjtzE4GDweG/6J/SDYV/ADFN/RLWGb+5rQ8wMCjc/fODgLJDFxk1Fwk/TTqTcbtLab1toLcts3zGIW5DstA3RQ0CCX/sPew5m7vh7DcKXluj2TBd9hw== 13241153187@163.com

进入GitHub个人的Settings -> SSH and GPG keys中，点击New SSH key，将复制的内容粘贴进去并保存。
使用如下命令检测是否成功（不用改代码）：
ssh -T git@github.com
此时会提示输入密码，输入前面的密码，回车即可，结果如下：

Enter passphrase for key '/c/Users/用户/.ssh/id_rsa':
Hi LeanGreen! You've successfully authenticated, but GitHub does not provide shell access.

下一步是设置用户名和邮箱:
git config --global user.name "your_name" //设置用户名
git config --global user.email "your_email@email.com" //设置邮箱
可通过命令查看设置情况：
$ git config -l

到此Git客户端已安装及GitHub配置完成，现在可以从GitHub传输代码了。

另外，还可以设置Git本地项目开发库默认路径
如果设置了，就不用每次打开Git再cd打开目录了。方法：右键Git Bash，找到快捷方式-起始位置，把你的项目地址放在这里就可以了。

3. 安装Hexo

首先，打开Git命令行，执行如下命令后关闭：
$ npm install -g hexo

然后在电脑中建立一个文件夹用于存放建立网站所需要的文件，如blog-z，并在此文件夹上右键打开Git Bash，执行：
$ hexo init
$ npm install
便可以自动在目标文件夹建立网站所需文件。

继续执行：
$ hexo server
